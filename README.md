# Программа учёта сходов с линии пассажирского транспорта

Локальная программа для фиксации, учёта и анализа сходов автобусов с линии.
Работает на вашем компьютере, открывается через браузер. Данные хранятся
в файле `data/incidents.db` (никуда в интернет не отправляются).

## Что умеет программа

- Загружать файл со сходами (`.xlsx`, `.xls`, `.csv`) — сама находит заголовки
  и распознаёт колонки, даже если названия немного отличаются.
- Автоматически определять причину и категорию поломки по тексту.
- Вести журнал сходов с поиском, фильтрами, редактированием и удалением.
- Вести справочник «База основных поломок» (можно дополнять и менять).
- Загружать справочники **водителей**, **диспетчеров** (Ф.И.О. + табельный номер)
  и **транспортных средств** (марка + госномер + класс автобуса) из файла Excel/CSV.
  Раздел «👥 Справочники» — все справочники можно **редактировать вручную** прямо
  в таблице: добавлять, изменять и удалять строки.
- Строить аналитику: по дням, маршрутам, автобусам, водителям, поломкам.
- Находить повторяющиеся неисправности и проблемные автобусы.
- Проверять качество данных и находить дубликаты.
- Формировать красивый Excel-отчёт для руководителя и технической службы.
- **Вход по логину и паролю** с разграничением прав по ролям.
- **Журнал действий** (аудит) и **резервные копии** базы.
- **Работа через интернет**: можно развернуть на сервере и открывать по адресу
  сайта (см. `DEPLOY_BEGET.md`).

## Вход и роли

При запуске открывается страница входа. При первом запуске создаётся
администратор (логин/пароль из `.env`, по умолчанию `admin`/`admin` —
**сразу смените пароль** в разделе «🔐 Пользователи»).

Роли и доступ:

| Роль | Что может |
|---|---|
| Администратор | всё + пользователи, настройки, справочники, резервные копии, аудит |
| Диспетчер | добавление/редактирование сходов, загрузка файлов, справочники, аналитика |
| Техническая служба | просмотр поломок, проблемных автобусов, повторов, контроля данных |
| Руководитель | аналитика и Excel-отчёты |
| Просмотр | только чтение |

## Как запустить (самый простой способ)

1. Откройте папку `line_incidents_app`.
2. Дважды кликните по файлу **`start.bat`**.
3. При первом запуске программа сама установит нужные библиотеки
   (нужен интернет, займёт 1–2 минуты).
4. Откроется браузер с программой. Если не открылся — зайдите на адрес,
   который покажет чёрное окно (обычно `http://localhost:8501`).

Чтобы закрыть программу — просто закройте чёрное окно.

## Как запустить вручную (для опытных)

```bash
pip install -r requirements.txt
streamlit run app.py
```

## Ярлык на рабочий стол

Кликните по `start.bat` правой кнопкой → «Отправить» → «Рабочий стол
(создать ярлык)». Ярлык можно переименовать, например «Сходы с линии».

## Как пользоваться

1. Раздел **«📥 Загрузка файла»** — загрузите Excel со сходами.
2. Программа распознает данные и классифицирует причины.
3. Проверьте нераспознанные причины в **«✅ Контроль данных»**.
4. Смотрите аналитику в разделах «Анализ по …».
5. Раздел **«📊 Excel-отчёт»** — кнопка «Сформировать отчёт» → скачивание
   готового файла `Отчёт_по_сходам_с_линии.xlsx`.

## Запуск на сервере (через интернет)

Чтобы программа была доступна по адресу сайта из любого места:

1. Установите на сервере Docker.
2. Скопируйте `.env.example` в `.env` и задайте пароль администратора.
3. Запустите: `docker compose up -d --build`.
4. Откройте `http://адрес-сервера`.

Подробная пошаговая инструкция (VPS, домен, HTTPS, бэкапы, обновление) —
в файле **`DEPLOY_BEGET.md`**.

## Структура проекта

```
line_incidents_app/
├── app.py                 — основной файл программы
├── config.py              — чтение настроек из .env
├── start.bat              — запуск двойным кликом (Windows)
├── requirements.txt       — список библиотек
├── README.md              — эта инструкция
├── DEPLOY_BEGET.md        — развёртывание на сервере
├── Dockerfile             — образ приложения
├── docker-compose.yml     — запуск приложения + Nginx
├── .env.example           — пример настроек
├── .streamlit/config.toml — настройки Streamlit
├── nginx/nginx.conf       — конфигурация Nginx
├── data/incidents.db      — база данных (создаётся автоматически)
├── uploads/               — загруженные файлы
├── reports/               — сохранённые отчёты
├── backups/               — резервные копии базы
└── modules/               — модули программы
    ├── database.py            — база данных SQLite
    ├── auth.py                — вход, пользователи, роли, права
    ├── audit.py               — журнал действий
    ├── backup.py              — резервные копии
    ├── import_data.py         — загрузка и распознавание файла
    ├── clean_data.py          — контроль качества и дубли
    ├── classify_breakdowns.py — автоклассификация причин
    ├── analytics.py           — аналитика
    └── excel_report.py        — формирование Excel-отчёта
```
